# 产品文案规范（Copy）

> 状态：草案 v0.1  
> 相关：[UX 索引](./README.md) · [本地化](./localization.md) · [交互](./interaction.md) · [文档写法](../writing.md)  
> 参考：[Apple HIG · Writing](https://developer.apple.com/design/human-interface-guidelines/writing)（气质对齐；中文落地以本文为准）

用清晰、简短、可行动的中文，写界面上每一个用户可见的词——按钮、标题、空状态、错误、确认框与设置说明。

---

## 概览

产品文案是交互的一部分：用户靠文字知道「现在怎样、能做什么、出了什么问题」。NextPili 的文案气质对齐 Apple HIG Writing 三原则，并结合中文产品习惯与 B 站社区用语。

| 原则 | 含义 | 检验 |
|------|------|------|
| **清晰（Clear）** | 用用户能懂的词；一次只说一件事 | 不查词典也能行动 |
| **简练（Concise）** | 删掉不改变意义的字 | 按钮 ≤ 4 字优先；提示一句说完 |
| **有用（Useful）** | 说明状态 + 下一步 | 错误态必有可执行动作或去向 |

| 边界 | 归属 |
|------|------|
| 界面用词、语气、术语、场景句式 | **本文** |
| ARB 键名、复数、RTL、格式化 | [localization.md](./localization.md) |
| 何时弹 Dialog / Snackbar | [interaction.md](./interaction.md) |
| 实现文档怎么写 | [writing.md](../writing.md) |

> **Important:** 用户可见字符串必须走 l10n；本文规定**写什么、怎么写**，不规定 ARB 工程细节。

## 了解基础知识

### 声音与人称

| 用 | 不用 |
|----|------|
| 第二人称或直接祈使：「检查网络后重试」 | 「用户需要检查网络」 |
| 冷静、直接、礼貌但不客套 | 卖萌、夸张、网络梗堆砌 |
| 主动语态：「无法加载评论」 | 「评论加载失败了呢…」 |
| 系统中性：「无法连接服务器」 | 指责用户：「你操作有误」 |

产品**不扮演**人格化客服；少用「我」「我们」指应用本身。

| | 示例 |
|--|------|
| 好 | 登录后即可发送弹幕 |
| 差 | 亲，登录一下我们才能帮你发弹幕哦～ |
| 好 | 无法播放此视频 |
| 差 | 呜呜播放失败了，再试试看嘛！ |

### 中文标点与语气

| 规则 | 说明 |
|------|------|
| 句号 | 完整说明句用「。」；按钮、菜单项、列表标题**不加**句号 |
| 感叹号 | 默认不用；仅成功轻提示可选「已收藏」（无感叹号更佳） |
| 问号 | 仅真正提问的确认框标题/正文 |
| 省略号 | 进行中用「加载中…」（三点）；不用「。。。」或英文 `...` 混排 |
| 引号 | 强调专名或用户输入时用「」 |
| 英文 | 专有名、快捷键、格式保留英文：`BV1…`、`Ctrl+Enter`、`4K` |

### 大小写与数字（中英混排）

| 场景 | 约定 |
|------|------|
| 中文界面主文案 | 中文为主；嵌入英文专名保持官方写法（Bilibili API 名不对用户暴露） |
| 英文界面（若有） | 按钮与菜单用 **Title Case** 或 **sentence case** 全文统一；NextPili 英文稿默认 **sentence case** 正文、按钮 **Title Case** 短标签 |
| 数字 | 用半角 `0-9`；计数缩写见 [localization.md](./localization.md) |
| 单位 | 与数字间中文习惯可无空格：`1080P`、`30 分钟`（英文数字与单位间一空格） |

### 术语（必须一致）

完整表维护在 [localization.md · 术语表](./localization.md#34-术语表节选可扩展)。产品文案**禁止**同义乱换：

| 统一用 | 不要混用 |
|--------|----------|
| 登录 / 退出登录 | 登入、注销、签出（设置项统一「退出登录」） |
| 稍后再看 | 稍后再看列表、待看（UI 标题用全称） |
| 清晰度 | 画质、分辨率（设置内可括号补充） |
| 弹幕 | 弹幕评论、滚动评论 |
| UP 主 | 上传者、作者（列表副文案可用「UP」缩写） |
| 分 P | 分集（仅番剧/剧集语境用「集」） |
| 重试 | 再试一次、刷新试试（同一按钮位固定「重试」） |

B 站社区词（弹幕、硬币、番剧、动态）中文内**保留**；勿换成生硬直译。

## 构建文案

按界面类型写。每条规则带 ✅ / ❌ 对照。

### 按钮与菜单

模式：**动词** 或 **动词 + 宾语**；描述点击后发生的事。

| | 文案 |
|--|------|
| ✅ | 登录、发送、收藏、重试、删除、退出登录 |
| ✅ | 清除观看历史、复制链接、添加到稍后再看 |
| ❌ | 确定、好的、提交（指代不明时） |
| ❌ | 点击登录、请按此按钮删除 |

| 规则 | Example |
|------|---------|
| 主按钮说结果 | 对话框主操作：「删除」；取消：「取消」 |
| 危险操作点名后果 | 主按钮：「退出登录」而非「确定」 |
| 成对按钮 | 左/次要「取消」；右/主要动作动词 |
| 加载中 | 按钮文案可改为「发送中…」或保留原文 + loading 指示；二者选一，勿叠两套「请稍候」 |

```text
✅  [ 取消 ]  [ 退出登录 ]
❌  [ 否 ]    [ 是 ]
❌  [ 取消 ]  [ 确定 ]
```

破坏性确认的主按钮必须复述动作，避免「是 / 确定」。

### 导航标题与设置

| 类型 | 句式 | Example |
|------|------|---------|
| 页面标题 | 名词或短名词短语 | 历史记录、设置、稍后再看 |
| 设置分组 | 名词 | 播放、弹幕、网络 |
| 设置项标题 | 名词或动宾 | 默认清晰度、自动连播 |
| 设置说明 | 一句完整结果 | 打开后，合集将在当前分 P 结束后自动播放下一项 |

说明文案写**行为结果**，不写实现细节（「写入本地偏好」「调用 playurl」）。

### 空状态

模式：**发生了什么** + **可以怎么做**（可有主按钮）。

| | 文案 |
|--|------|
| ✅ | 暂无观看历史<br>看过的视频会出现在这里 |
| ✅ | 收藏夹是空的<br>浏览时点星标即可收藏 |
| ❌ | 空空如也～ |
| ❌ | 没有数据 |
| ❌ | 列表为 null |

登录门闸空态同样遵守：说明价值，不施压。

```text
✅  登录后同步收藏与稍后再看
    [ 登录 ]

❌  你还没有登录，无法使用本功能！！
    [ 立即登录]
```

### 成功与轻反馈（Toast / Snackbar）

| 规则 | ✅ | ❌ |
|------|----|----|
| 已完成、可撤销 | 已取消收藏 · 撤销 | 操作成功 |
| 已复制 | 链接已复制 | 复制成功！ |
| 尽量可感知对象 | 已添加到稍后再看 | 成功 |

不弹成功 Toast 的情况：导航切换、播放/暂停、纯 UI 展开折叠——**反馈靠界面状态**，不靠文案刷屏。

### 进行中

| 场景 | 文案 |
|------|------|
| 首屏/列表 | 加载中… |
| 播放地址 | 正在获取播放地址… |
| 登录 | 登录中… |
| 发送 | 发送中… |

同一流程只用一种进行中说法；避免「请稍候」「Loading」「加载中」混用。

### 错误与恢复

对齐 Apple：说明问题、避免指责、给出下一步。

模式：**原因（可读）** + **下一步（可点）**。

| | 文案 |
|--|------|
| ✅ | 网络不可用，请检查连接后重试<br>[ 重试 ] |
| ✅ | 登录已失效，请重新登录<br>[ 登录 ] |
| ✅ | 无法播放此视频，可尝试切换清晰度<br>[ 切换清晰度 ] [ 重试 ] |
| ❌ | 错误 -101 |
| ❌ | Request failed |
| ❌ | 你的网络有问题 |
| ❌ | 未知错误，请联系管理员 |

| 错误类型 | 用户可见策略 |
|----------|----------------|
| 网络 / 超时 | 本地化模板 + 重试 |
| 未登录 `-101` 等 | 门闸文案 + 登录，不贴原始 JSON |
| 权限 / 地区 | 明确不可用原因；**不提供**绕过暗示 |
| 服务端 `message` | 可作调试详情；默认 UI 用稳定错误码映射句 |
| 未识别错误 | 「出了点问题，请稍后重试」+ 重试；高级入口可看详情 |

```text
Rust AppError.kind  →  l10n.errorNetworkUnavailable
显示：「网络不可用，请检查连接后重试」
不显示：HTTP 500 body / 堆栈
```

### 确认与危险操作

| 要素 | 要求 | Example |
|------|------|---------|
| 标题 | 点名动作或后果 | 退出登录？ / 删除观看历史？ |
| 正文 | 不可逆后果一句说清 | 退出后，本机将停止同步收藏与稍后再看。 |
| 主按钮 | 动词复述 | 退出登录 / 删除 |
| 次按钮 | 取消 | 取消 |

```text
✅  标题：清除观看历史？
    正文：此操作不可撤销。
    按钮：取消 · 清除

❌  标题：提示
    正文：是否确认？
    按钮：取消 · 确定
```

### 权限与系统能力

说明**为何需要**，再请求；拒绝后给设置去向（若平台允许）。

| ✅ | ❌ |
|----|----|
| 需要通知权限，以便在下载完成时提醒你 | 请允许通知 |
| 无法访问剪贴板时：改为手动复制说明 | 权限被拒绝 |

### 无障碍文案

- `Semantics` / Tooltip 写**控件职责**，不写视觉外观：「播放」「暂停」「下一分 P」。
- 避免「蓝色按钮」「右上角图标」。
- 整句进 ARB，禁止碎拼接（见 [localization.md](./localization.md)）。

## 场景速查

### 认证

| 场景 | 推荐文案 |
|------|----------|
| 登录按钮 | 登录 |
| 扫码等待 | 请使用手机客户端扫码 |
| 扫码过期 | 二维码已过期<br>[ 刷新 ] |
| 退出确认标题 | 退出登录？ |
| 会话失效 | 登录已失效，请重新登录 |

### 播放器

| 场景 | 推荐文案 |
|------|----------|
| 播完下一 P | 即将播放下一分 P<br>[ 取消 ] |
| 清晰度 | 自动 / 1080P / 4K（与数据层 qn 映射表一致） |
| 弹幕关闭 | 弹幕已关闭（可选 Toast；状态以图标为准） |
| 发送失败·未登录 | 登录后即可发送弹幕 |
| 地址失效 | 播放地址已失效，正在重新获取… → 失败则「无法获取播放地址」+ 重试 |

### 列表与内容

| 场景 | 推荐文案 |
|------|----------|
| 加载失败条 | 加载失败<br>[ 重试 ] |
| 无搜索结果 | 未找到相关内容<br>试试其他关键词 |
| 已复制链接 | 链接已复制 |
| 已收藏 / 取消 | 已收藏 · 已取消收藏（可带撤销） |

## 深入约束

### 禁词与禁句式

| 禁止 | 原因 | 替换 |
|------|------|------|
| 操作成功 / 操作失败 | 无信息 | 说具体对象与结果 |
| 请稍候（单独、无限期） | 无进度、无结束 | 加载中… / 带超时重试 |
| 哎呀 / 呜呜 / 亲 / 小主 | 声音漂移 | 直接陈述 |
| 你错了 / 非法操作 | 指责用户 | 说明系统侧限制 |
| 未知错误（无动作） | 死胡同 | 通用失败句 + 重试 |
| 点击这里 | 无障碍与扫描差 | 直接写动作名 |
| Lorem / 测试文案进正式包 | 质量事故 | ARB 正式句 |

### 幽默与品牌

- **空状态**可轻度轻松，仍须有用。
- **错误、支付、权限、清数据**保持严肃。
- 不模仿 B 站官方营销口吻；不使用 B 站粉相关品牌口号。
- 不承诺「官方」「更快破解」等违规或误导表述。

### 长度预算（桌面优先）

| 槽位 | 建议上限（中文） | 溢出策略 |
|------|------------------|----------|
| 按钮 | 2–6 字 | 缩短动词；勿缩成难懂两字黑话 |
| Rail 标签 | 2–4 字 | 全名放 Tooltip |
| Toast | 1 句，≤ 20 字 | 分主次，勿堆两句 |
| 空状态标题 | ≤ 12 字 | 说明放下文 |
| 对话框正文 | 1–3 句 | 超过则列要点 |

长语言（英文/德文）布局预留见 [localization.md](./localization.md) 与 [multi-platform.md](./multi-platform.md)。

### 与文档用语的分工

| | 产品 UI（本文） | 工程文档（writing.md） |
|--|-----------------|------------------------|
| 读者 | 终端用户 | 实现者 |
| 术语 | 社区习惯 + 术语表 | crate / API 符号 |
| 错误 | 可行动中文 | `code` → `Error` 映射表 |
| 「你」 | 使用产品的人 | 写代码的人 |

## 提交前自检

- [ ] 导语/按钮是动词或结果，不是「确定/提示」
- [ ] 错误有原因 + 下一步；无原始码/堆栈直出
- [ ] 危险确认主按钮复述动作
- [ ] 术语与术语表一致；无同义混用
- [ ] 无硬编码（走 ARB）；无禁词表用语
- [ ] 空状态说明价值与下一步
- [ ] Tooltip / Semantics 描述职责而非外观
- [ ] 中文标点符合本节；进行中用「…」
