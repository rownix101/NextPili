# 人机交互（Interaction）

> 状态：草案 v0.1  
> 依赖：[UX 索引](./README.md) · [设计规范](./design-system.md) · [动效](./motion.md) · [多平台](./multi-platform.md)

本文约定信息架构、导航、输入方式、播放器交互、反馈与错误处理。桌面优先，同时预留触摸与移动语义。

---

## 1. 信息架构

### 1.1 一级导航（建议）

| 入口 | 内容 |
|------|------|
| 首页 | 推荐 / 热门 / 分区入口 |
| 动态 | 关注动态（需登录） |
| 媒体库 | 历史、收藏、稍后再看、下载（若有） |
| 直播 | 直播推荐与分区（可并入首页 Tab） |
| 消息 | 首期可弱化或入口放「更多」 |
| 我的 / 账号 | 登录态、设置、多账号 |

搜索为 **全局入口**（顶栏），不单占一级 Tab。

### 1.2 核心用户路径

```text
发现（推荐/热门/搜索）
  → 视频详情 / 直接播放
    → 播放控制（清晰度、弹幕、字幕、倍速、全屏）
    → 评论 / 相关推荐
    → 关注 UP、收藏、分享（系统分享/复制链接）

登录（可选阻断）
  → 收藏夹 / 历史同步 / 发送评论弹幕
```

设计优先级：**播放路径最短**；登录仅在写操作或个性化必要处引导。

### 1.3 页面类型

| 类型 | 示例 | 交互特征 |
|------|------|----------|
| 浏览型 | 推荐网格、搜索结果 | 无限滚动、筛选、虚拟列表 |
| 消费型 | 播放器页 | 快捷键、自动隐藏控件、沉浸 |
| 阅读型 | 动态、评论 | 分页/楼中楼、输入 |
| 配置型 | 设置 | 表单、即时保存或显式保存需统一 |

## 2. 导航模式

### 2.1 桌面默认

```text
┌──────┬─────────────────────────────┬────────────┐
│ Rail │  顶栏（搜索 · 账号 · 窗口）   │            │
│ 导航 │─────────────────────────────│  可选右侧  │
│      │  主内容（可含二级 Tab）       │  详情/列表 │
│      │                             │            │
└──────┴─────────────────────────────┴────────────┘
```

- **NavigationRail**：一级导航；可折叠为仅图标。
- **顶栏**：搜索、全局操作；不重复一级导航。
- **二级**：`TabBar` / `SegmentedButton`（推荐 | 热门 | 排行）。
- **深层页**：内容区 `Navigator` 栈；系统返回 / 顶栏返回 / 鼠标侧键（若平台支持）。

### 2.2 窄窗口

- Rail → **NavigationBar** 或折叠 Drawer。
- 播放页优先占满；评论可转路由页或底部 sheet（桌面更推荐分栏/下页）。

### 2.3 路由与深链

- 路由可表达：视频 `bvid`/`aid`、播放页、用户空间、搜索词。
- 桌面支持从系统打开 URL（后续）；UI 需能根据 route 重建状态。
- 返回栈：播放中途返回保留列表滚动位置（状态恢复）。

## 3. 输入范式

### 3.1 指针（鼠标）

| 行为 | 约定 |
|------|------|
| 左键 | 主操作（打开、播放、点击按钮） |
| 右键 | 上下文菜单（视频卡、评论、播放器） |
| 中键 | 可选：新标签/新窗口打开视频（桌面增强，非必须首期） |
| Hover | 视频卡快捷操作、播放器显示控件 |
| 滚轮 | 列表滚动；播放器上可配置为音量或时间（默认音量更常见，需设置项） |
| 拖拽 | 进度条拖拽 seek；窗口拖放链接打开（可选） |

### 3.2 触摸板 / 触摸

| 手势 | 场景 | 行为 |
|------|------|------|
| 点击 | 通用 | 同左键 |
| 双击 | 播放器 | 全屏切换 **或** 左右区域快进退（需二选一并写死，避免冲突） |
| 左右滑 | 播放器 | seek（可选，默认关闭以免误触） |
| 上下滑 | 播放器 | 音量/亮度（移动向）；桌面默认不用 |
| 长按 | 列表项 | 进入多选或同右键菜单 |
| 捏合 | 播放器 | 可选：画面缩放（看情况） |

**双击策略（建议拍板）**：

- 桌面默认：**双击切换全屏**；单击播放器中央切换播放/暂停。
- 左右双击 ±10s 作为可选增强（设置开关），默认关，减少与全屏冲突。

### 3.3 键盘（全局与播放器）

全局（非输入框焦点时）：

| 快捷键 | 动作 |
|--------|------|
| `/` 或 `Ctrl+K` | 聚焦搜索 |
| `Ctrl+1…9` | 切换一级导航（可选） |
| `Esc` | 关闭对话框 / 退出全屏 / 清除焦点层 |
| `?` | 快捷键帮助叠层 |

播放器（焦点在播放器或播放页时）：

| 快捷键 | 动作 |
|--------|------|
| `Space` / `K` | 播放 / 暂停 |
| `J` / `L` | 后退 / 前进 10s |
| `←` / `→` | 后退 / 前进 5s |
| `↑` / `↓` | 音量 ± |
| `M` | 静音 |
| `F` | 全屏 |
| `T` | 剧场/最大内容宽（可选） |
| `C` | 弹幕开关 |
| `S` | 字幕开关（若有） |
| `>` / `<` | 倍速升/降 |
| `0…9` | 跳到时间轴 0%…90% |
| `[` / `]` | 上一集 / 下一集（合集/番剧） |
| `Esc` | 退出全屏 |

约定：

- 输入框/评论获焦时，**不**触发播放器字母键（Space 在多行输入中为换行）。
- 快捷键表在设置与 `?` 面板展示，并支持后续自定义（首期可写死）。
- 与系统全局快捷键冲突时，应用内可降级，不强制劫持系统键。

### 3.4 游戏手柄 / 媒体键

- 系统媒体键（播放暂停、下一首）：若 OS/播放器插件可达，应对接；首期可为 nicetohave。
- 不做游戏手柄为默认支持。

## 4. 播放器交互

### 4.1 控件布局（桌面）

```text
┌──────────────────────────────────────────────────────────┐
│ 标题 · UP · 关注                              [PiP] [×]  │  顶栏（hover/暂停时显示）
│                                                          │
│                     （视频画面）                           │
│                                                          │
│  ▶  0:12 / 12:34  ──●──────────  弹幕  清晰度  倍速  全屏 │  底栏
└──────────────────────────────────────────────────────────┘
```

- 控件 **自动隐藏**：播放中无操作约 **2.5–3s** 后淡出；移动鼠标/按键再显示。
- 暂停时：**保持显示**控件。
- 进度条：**hover 放大**命中区域；拖拽时显示预览时间（缩略图可选后续）。
- 点击进度条：seek 到位置；键盘微调见上表。

### 4.2 清晰度 / 线路 / 音轨

- 入口在底栏菜单；切换时：
  - 尽量 **无缝或短缓冲**；
  - Toast/轻提示当前清晰度；
  - 失败回退上一档并说明原因。
- 默认清晰度：用户设置 > 上次选择 > 自动（带宽启发式，后续）。

### 4.3 弹幕

| 操作 | 行为 |
|------|------|
| 开关 | `C` 或底栏按钮；状态持久化 |
| 不透明度 / 密度 / 速度 / 屏蔽词 | 设置面板或播放器二级菜单 |
| 发送 | 输入框；登录门闸；频率限制时友好提示 |
| 碰撞与可读 | 描边/阴影保证浅色画面可读；屏蔽色块弹幕可选 |

弹幕层 **不拦截** 播放器主点击（除弹幕输入聚焦时）。

### 4.4 全屏 / 剧场 / 画中画

| 模式 | 行为 |
|------|------|
| 全屏 | 占满当前屏幕；Esc/`F` 退出；系统全屏 API |
| 剧场 | 隐藏侧栏，加宽播放器（桌面） |
| 画中画 | OS PiP 若可用；否则应用内小窗（后续） |
| 多显示器 | 全屏落在播放器所在屏 |

### 4.5 合集 / 多 P / 番剧

- 清晰展示当前 P 与列表；支持自动连播（设置项，默认开或关需产品拍板，建议 **合集默认开、推荐页默认关**）。
- 结束后：下一 P 倒计时可取消。

### 4.6 错误与恢复

| 情况 | 交互 |
|------|------|
| 网络中断 | 保留进度；显著重试；恢复后可续播 |
| 地址过期 | 静默刷新 playurl，失败再提示 |
| 地区/权限 | 明确文案，不提供绕过暗示 |
| 解码失败 | 提示切换清晰度或检查硬件加速设置 |

## 5. 列表与 ent 交互

### 5.1 信息流

- 滚动加载：距底部阈值触发下一页；失败可点重试条。
- 下拉刷新：触摸场景；桌面可用顶栏刷新按钮 / `Ctrl+R`（若与系统刷新不冲突）。
- 筛选与排序：变更后列表回顶。

### 5.2 搜索

- 输入防抖 **200–300ms** 后再请求建议（若有建议 API）。
- Enter 提交完整搜索；历史记录本地保存（非敏感）。
- 结果页：综合 / 视频 / 用户等 Tab。

### 5.3 评论

- 楼中楼展开折叠；点赞需登录。
- 桌面：`Ctrl+Enter` 发送，`Enter` 换行（或相反，**必须在设置中写清默认**；建议 **Enter 换行，Ctrl+Enter 发送**）。
- 排序：热度 / 时间。

## 6. 反馈与系统状态

### 6.1 反馈层级

| 层级 | 组件 | 场景 |
|------|------|------|
| 即时 | 涟漪/hover/按压缩放 | 点击可感知 |
| 轻量 | Tooltip、按钮 loading | 悬停说明、提交中 |
| 中间 | Snackbar / Toast | 已收藏、已复制、可撤销删除 |
| 阻断 | Dialog | 丢弃未保存、退出登录、权限 |

原则：**能非阻断就不弹窗**；破坏性才确认。

### 6.2 撤销

- 收藏移除、稍后再看移除等：Snackbar **带撤销**，窗口期 **3–5s**。
- 无法撤销的操作：对话框明确后果。

### 6.3 剪贴板与分享

- 复制链接：Toast「已复制」。
- 系统分享面板：移动向；桌面以复制为主。

## 7. 焦点、语义与无障碍

- 所有可点击控件可获焦；Tab 顺序符合视觉顺序（播放器内控件有合理序列）。
- `Semantics`：图标按钮提供 label（「播放」「静音」「全屏」）。
- 动画：遵循「减少动态效果」系统设置（见 motion）。
- 不要仅用颜色表示状态（如仅红色表示错误）。
- 字幕与弹幕：可调字号；冲突时用户可选隐藏其一。

## 8. 权限与登录门闸

| 操作 | 未登录行为 |
|------|------------|
| 浏览推荐/搜索/播放 | 允许（以 API 能力为准） |
| 点赞/收藏/发评/发弹幕/历史同步 | 引导登录（对话框或内联 banner） |
| 多账号切换 | 设置/账号菜单 |

登录流程本身的交互遵循 API 文档（二维码/密码等），UI 保持步骤清晰、可取消、错误可读。

---

## 9. 性能相关交互约定

- 列表快速滑动时：**降低**非关键动画与图片解码优先级。
- 页面切换：优先展示缓存/骨架，避免白屏闪烁。
- 播放中后台标签页：可降级帧率或暂停（策略后续与媒体层对齐）。

---

## 10. 验收清单（交互）

- [ ] 播放路径 ≤ 2 次主要点击（从首页卡片到出画）
- [ ] 键盘可完成：搜、播、暂停、seek、全屏、关弹幕
- [ ] 输入框获焦时快捷键不误触播放
- [ ] 右键菜单覆盖视频卡核心操作
- [ ] 错误态可恢复；登录门闸文案明确
- [ ] 控件自动隐藏不与拖拽进度条冲突
